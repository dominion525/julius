Julius最新版

Unix版 (オリジナル版) Juliusの最新アーカイブです．

質問等はユーザフォーラムへどうぞ.

動作環境はこちらで確認できます．

Julius/Julianには「通常仕様」と「マルチパス仕様」の 2 つのタ イプがあります． 「マルチパス仕様」は仕様拡張により，複雑な遷移を持つ音響モデルを扱うことが できますが，その分処理速度は若干遅くなります．

開発版スナップショット

開発中のコードをJulius開発サイト上で公開しています． 最新のバグ修正やテスト中の追加機能など，最新のコードが随時公開されています．

開発中のコードはどなたでも入手可能です．アクセス・入手方法は以下の通りです．

なお，Julius-4の開発版 CVS レポジトリは新たに julius4 と なりました．下記のリンクは最新版を指しています．従来のjulius には 3.x が入っていますのでご注意下さい．

  • ブラウザで閲覧

    Web ブラウザで SourceForge.jp: CVS Repository: julius4 のページを開いて下さい．コードツリーを直接閲覧できます．リ ク先では，ファイルごとの変更履歴の確認や過去のバージョン からの差分チェックも行えます．

  • アーカイブとして一括ダウンロード

    コード全体を一括ダウンロードするには，上記と同じ SourceForge.jp: CVS Repository: julius4 へ行き，そのページの最下部にある 「Download tarball」をクリックしてください．ソースツリー全体が tar.gz 形式で圧縮されてダウンロードされます．

  • CVS で取得

    Anonymous CVS を用いてソースコードをダウンロードできます． CVS を利用することで，サーバ側とローカルでバージョン管理が 行われるので，あとでJuliusが更新されたときに，サーバから更 新差分だけをダウンロードするといったことができます．例えば， Linux や cygwin で CVS 経由で取得するには，コマンドライン から以下のように cvs を実行します．

    % cvs -z3
      -d:pserver:anonymous@cvs.sourceforge.jp:/cvsroot/julius co
      julius4

    一度取得したあとは，以下のコマンドで，そのディレクトリより 下にあるファイルについて，サーバー上の更新をローカルに反映させる ことができます．

    % cvs update -Pd
    ※ なお， julius-cvs@lists.sourceforge.jp に登録することで，CVS上のソースの更新通知をその都度メールで受け 取ることができ ます．どなたでも登録できますので，最新の更新状況に興味のあるかた は，julius-cvs@lists.sourceforge.jpの登録ページからどうぞ．

Rev. 4.0 (2007/12/19)

[リリースノート]

Julius-4.0 は，拡張性や柔軟性の確保を目指し，大幅な刷新が行われました． Juliusの内部データ構造の大幅な見直しおよびモジュール化を行った結果，エ ンジン本体のライブラリ化やマルチデコーディングなど多くの先進的機能を実 現しました．主な新機能は以下のとおりです（全てではありません）．

  • エンジン本体のライブラリ化 "JuliusLib"
  • 複数モデルを用いたマルチデコーディングの対応
  • Julius / Julian の一本化
  • 言語モデルや認識処理の動的追加・削除が可能
  • 4-gram 以上の長さのN-gramの対応
  • ユーザ定義言語関数の組込み
  • Confusion network 出力
  • 音声区間検出 (VAD) の強化（GMM・デコーダベース）
  • ツールの機能追加・新ツール追加・メモリ管理改善等

上位互換性を確保しており，Julius-3.x と同様に使うことが出来ます． 認識の精度および速度は，現段階では前バージョン (3.5.3) とほぼ 同じです．以下のドキュメントも合わせて御覧下さい．

  • Rev.4.0 の新機能
  • JuliusLib API リファレンス
  • JuliusLib コールバック一覧
  • Juliusコマンドマニュアル
  • jconf設定ファイルマニュアル

Rev. 3.5.3 (2006/12/29)

[リリースノート｜ChangeLog]

Julius-3.5.3 では，パフォーマンスの改善，文法関連ツールの追加， 単語グラフの改善やバグ修正が行われました．主な変更点は以下の通りです．

  • 高速化：コード最適化により処理速度が 1.1倍〜1.5倍に高速化．
  • 音響分析条件の設定方法を拡張
    • HTK Config を直接読み込めるようになった (-htkconf)
    • バイナリHMMへ設定を埋め込めるようになった
  • 新たな文法最適化ツール：dfa_minimize, dfa_determinize
  • その他の機能追加：
    • 単語グラフ生成で左右のコンテキストごとに異なる仮説の生成をサ ポート
    • オンラインでのエネルギー項正規化の暫定サポート
  • バグ修正およびコードの改善
